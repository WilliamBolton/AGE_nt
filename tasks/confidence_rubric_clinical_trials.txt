GOAL
Return a confidence score (0–100) that the intervention has beneficial effects on ageing/healthspan/longevity in humans,

EVIDENCE GATHERING STRATEGY (CLINICALTRIALS ONLY)
- Use summary_stats.by_source_type to confirm clinicaltrials availability.
- Query clinicaltrials records broadly first (no full-text query), then refine using clinicaltrials fields:

- Aggregate evidence across multiple trials; do not score from a single trial.

PER-DOCUMENT GRADING (for fetched clinicaltrials records)
For each clinicaltrials record you use in scoring, infer the following fields ONLY from scraped content:

A) Evidence level and design (use canonical hierarchy)
- level ∈ {1,2,3,4,5,6}
  - 1 = Systematic reviews & meta-analyses
  - 2 = Randomised controlled trials (RCTs), including phase trials
  - 3 = Observational / epidemiological studies
  - 4 = Animal model studies (in vivo)
  - 5 = Cell culture / in vitro studies
  - 6 = In silico / computational predictions
- In this CLINICALTRIALS-ONLY rubric, clinicaltrials records should be treated as level 2 evidence by default.
- Do not downgrade a clinicaltrials record to level 3/6 only because fields are sparse; reflect missingness in quality subscores instead (especially info_sufficiency/reporting_quality).
- design_label ∈ {rct, interventional_nonrandomized, clinical_trial_registry, other}
- human_flag: true
- source_type: "clinicaltrials"

B) Relevance to ageing/healthspan/longevity claims
- relevance ∈ {0,1,2}
  0 = not relevant to ageing/healthspan/longevity (e.g. unrelated disease population without ageing rationale)
  1 = indirectly relevant (surrogate biomarkers, adjacent indication, unclear ageing connection)
  2 = directly relevant (older adults, immune ageing, frailty, functional outcomes, clinically meaningful healthspan proxies, or explicitly ageing-targeted trial)

C) Quality subscores (0–2 each; do NOT invent missing info)
Score based ONLY on fields present in the record (title, design/arms, outcomes, results_summary, dates, enrollment, etc.)
- methods_clarity: is trial design described clearly (randomized/blinded/controls/arms)?
- bias_controls: randomization/blinding/comparator present?
- sample_adequacy: enrollment size adequate for design (0 tiny/unclear, 1 moderate, 2 adequate)
- reporting_quality: are outcomes/results/AEs described clearly (esp. if results are posted)?
- info_sufficiency: are key fields present (status, phase, outcomes, enrollment, results_summary)?

D) Outcome directness
- endpoint_grade ∈ {0,1,2,3}
  0 = no outcomes interpretable / mechanistic only
  1 = surrogate biomarkers only
  2 = validated functional/clinical proxies (frailty, infections, gait, immune function with clinical relevance)
  3 = hard endpoints (morbidity/mortality/disability-free survival, hospitalizations)

E) Direction of evidence (only if results are present)
- effect_direction ∈ {-1,0,+1}
  +1 = benefit signal
  0 = unclear/mixed/insufficient results
  -1 = harm/negative signal
- effect_strength ∈ {0,1,2} (weak/moderate/strong)
IMPORTANT: If results are not present in the record, set effect_direction=0 and effect_strength=0.

DOCUMENT WEIGHT (clinicaltrials-only; conceptual)
Base weight by level (canonical):
  level1 1.00
  level2 0.85

Multipliers:

  m_direction: +1->1.0, 0->0.85, -1->0.65
  m_strength: 0->0.85, 1->1.0, 2->1.1
doc_weight = w_level * m_relevance * m_quality * m_endpoint * m_direction * m_strength

AGGREGATION (clinical trials only)
Use canonical tier aggregation:
- For each level i in {1..6}, compute S_i = sum(doc_weight at level i)
- C_i = cap_i * (1 - exp(-S_i / tau_i))
- Caps: cap1=30, cap2=30, cap3=15, cap4=12, cap5=8, cap6=5
- Taus: tau1=1, tau2=2, tau3=3, tau4=5, tau5=7, tau6=8
- raw_confidence = sum(C_i for i=1..6)

In this clinicaltrials-only run, most contribution should typically come from level 2.

GATING CHECKLIST (clinicaltrials-only; scraped corpus only)
Compute booleans from your graded clinicaltrials docs:
1) any_human_evidence: true if >=1 clinicaltrials record has relevance>0
2) any_rct: true if >=1 level 2 doc has relevance>0
3) any_human_endpoints: true if any level 2–3 doc has endpoint_grade >=2
4) any_human_safety: true if any level 2–3 doc has reporting_quality >=1 AND mentions AEs/safety/tolerability (or explicit safety outcomes)

Penalties (multiply):
- If no human evidence: x0.0 (confidence must be 0)
- Else if human evidence but no RCT: x0.75
- If RCTs exist but no endpoint_grade>=2: x0.85
- If no safety reporting: x0.90
gating_penalty = product of applicable penalties
confidence = raw_confidence * gating_penalty
